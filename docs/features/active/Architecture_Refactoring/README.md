# Architecture Refactoring (아키텍처 리팩토링)

## 개요

ACS 프로젝트 전체 아키텍처 리팩토링 계획입니다. 프론트엔드와 백엔드의 코드 품질, 구조, 성능을 혁신적으로 개선합니다.

## 목표

1. **폴더 구조 정리**: 애매한 위치의 파일 정리, 중복 제거
2. **코드 중복 제거**: Settings 컴포넌트 통합, 공통 패턴 추출
3. **거대 파일 분해**: 300줄 이상 파일을 단일 책임 원칙에 맞게 분리
4. **성능 최적화**: 캐싱, 선택적 업데이트, 불필요한 재렌더링 제거
5. **테스트 커버리지**: 0% → 60% 이상

## 현재 상태

| 영역 | 파일 수 | 테스트 | 복잡한 파일 (300줄+) |
|------|--------|--------|---------------------|
| 백엔드 | 66개 | 1개 (1.5%) | 29개 |
| 프론트엔드 | 43개 컴포넌트, 25개 스토어 | 0개 (0%) | 18개 |

## 작업 단계

| Phase | 내용 | 리스크 |
|-------|------|--------|
| **Phase 0** | 폴더 구조 정리 (파일 이동, 중복 제거) | 🟢 낮음 |
| **Phase 1** | Settings Composable, 에러매핑, 로거 | 🟢 낮음 |
| **Phase 2** | icdStore 개선, SunTrackService 분해 | 🟡 중간 |
| **Phase 3** | EphemerisService 분해, 테스트 | 🔴 높음 |
