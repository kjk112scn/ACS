# Plan - 계획 수립

## 역할

작업 전 계획을 수립하여 방향을 명확히 합니다.

## 워크플로우

```
[1] 요청 분석
    └── 목표, 범위, 제약 파악

[2] 현황 조사
    └── 관련 코드/문서 확인

[3] 계획 수립
    └── 단계별 작업 목록

[4] 합의
    └── 사용자 확인 후 진행
```

## 계획 문서 형식

```markdown
# {작업명} 계획

## 목표
{달성하고자 하는 것}

## 현황 분석
- 관련 파일: {파일 목록}
- 현재 상태: {요약}

## 작업 계획

### Phase 1: {단계명}
- [ ] 작업 1
- [ ] 작업 2

### Phase 2: {단계명}
- [ ] 작업 3
- [ ] 작업 4

## 예상 영향
- {영향받는 컴포넌트/기능}

## 대안
| 방안 | 장점 | 단점 |
|------|------|------|
| A | | |
| B | | |

## 결정 필요 사항
1. {질문}
```

## 사용 예시

```
사용자: "/plan 로그인 기능 추가"

AI:
📋 요청 분석

**이해한 내용:**
- 로그인 기능 구현 필요
- 사용자 인증 시스템

**확인 필요:**
1. 인증 방식? (JWT / Session / OAuth)
2. 소셜 로그인 필요?
3. 회원가입도 포함?

어떻게 진행할까요?
```

## 복잡한 작업일 때

```
작업 규모가 커서 단계별로 나눠서 진행하는 게 좋겠어요.

**추천 순서:**
1. 기본 인증 API (2시간)
2. 로그인 폼 UI (1시간)
3. 상태 관리 연동 (1시간)
4. 테스트 작성 (1시간)

Phase 1부터 시작할까요?
```
