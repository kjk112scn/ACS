# Sync - 문서 동기화 스킬

## 역할

코드와 문서의 일관성을 유지하고 문서 품질을 관리합니다.

**핵심 가치:**
- 코드 변경 → 문서 자동 반영 제안
- 깨진 링크/참조 감지
- 문서 구조 일관성 유지

---

## 워크플로우

```
[1. 스캔] → [2. 비교] → [3. 감지] → [4. 수정] → [5. 보고]
     │          │          │          │          │
  docs/     코드 vs    불일치      자동/수동    결과
  구조       문서       목록       수정        요약
```

---

## 점검 항목

### 1. 링크 검증

```yaml
검사:
  - 마크다운 내부 링크 ([text](path))
  - 상대 경로 유효성
  - 앵커 링크 (#section)

출력:
  ❌ 깨진 링크: docs/guides/setup.md → docs/old/removed.md
  ✅ 유효한 링크: 42개
```

### 2. 코드↔문서 동기화

```yaml
검사:
  - context/ 문서가 실제 코드 구조 반영
  - 함수/클래스명 변경 시 문서 업데이트
  - API 엔드포인트 변경 감지

출력:
  ⚠️ 불일치 발견:
  - backend.md: "UserController" → 실제: "AuthController"
```

### 3. 문서 구조 일관성

```yaml
검사:
  - 필수 파일 존재 (README.md, PROGRESS.md)
  - 폴더 명명 규칙 (PascalCase_Underscore)
  - 상태 표기 일관성 (🚧, ✅, ⏳)

출력:
  ⚠️ 구조 문제:
  - work/active/login/ → Feature_Login/ 이어야 함
```

### 4. CURRENT_STATUS 일관성

```yaml
검사:
  - 진행 중 목록과 active/ 폴더 일치
  - 완료 목록과 logs/ 기록 일치
  - 진행률 표기 정확성

출력:
  ⚠️ 불일치:
  - CURRENT_STATUS: Feature_Login 진행 중
  - 실제: active/에 폴더 없음
```

---

## 실행 모드

### 기본 모드 (점검만)

```
/sync

→ 점검 결과만 출력, 수정 안 함
```

### 자동 수정 모드

```
/sync --fix

→ 안전한 수정 자동 적용:
  - 깨진 링크 제거/수정 제안
  - CURRENT_STATUS 업데이트
  - 누락된 필수 파일 생성
```

### 특정 영역 점검

```
/sync links      # 링크만 점검
/sync status     # CURRENT_STATUS만 점검
/sync structure  # 폴더 구조만 점검
```
